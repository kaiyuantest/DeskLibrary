# NextVault 增删改查示例

这份文档是给“进入数据目录后工作的外部 AI / Codex”用的。

核心数据目录：

`C:\Users\Asus\Documents\DeskLibrary.NextVault`

使用原则：

- 先查清目录和对象结构，再动数据。
- 默认只读，不修改。
- 只有我明确说“可以修改/删除/新增”时，才执行写操作。
- 每次写操作前，先告诉我将改哪些文件、改什么。
- 每次写操作后，告诉我实际改了哪些文件。

---

## 1. AI 第一次进入目录时的标准提示词

```text
你现在所在目录是 DeskLibrary 的新数据根目录 NextVault。
这里不是旧版 AppData 存储，而是新的可控数据仓库。

你的工作规则：
1. 先扫描目录结构和 JSON 文件，理解数据组织
2. 默认只读，不要修改任何文件
3. 当我要求你查数据时，直接基于这里的文件回答
4. 如果我要你改数据，你必须先说明将修改哪些文件，再执行
5. 执行后要汇报实际修改了哪些文件

你先告诉我：
- 这里有哪些主要子目录
- 文本、图片、文件、浏览器卡片分别在哪
- 你接下来应该如何查找数据
```

## 2. 查：查询类示例

### 2.1 查某段文字

```text
请在这个 NextVault 目录里查找包含“发票”“合同”“报价单”的文本内容。

要求：
1. 先查文本记录
2. 再查文档和备注字段
3. 返回命中的文件路径、对象类型、命中片段
4. 不要修改任何文件
```

### 2.2 查某个网站卡片

```text
请在这个 NextVault 目录里查找和 `example.com`、`google.com`、`github.com` 相关的浏览器卡片。

要求：
1. 查 domain、name、openUrl、remark、username
2. 告诉我命中的卡片有哪些
3. 返回对应文件路径
4. 不要修改数据
```

### 2.3 查某个文件资源

```text
请在这个 NextVault 目录里查找文件名、备注、路径里包含“项目计划”“Excel”“png”的资源。

要求：
1. 查 file asset
2. 告诉我是 link 还是 backup
3. 返回 sourcePath、storedPath、primaryPath
4. 不要修改任何文件
```

### 2.4 查最近新增了什么

```text
请总结这个 NextVault 里最近新增的数据。

要求：
1. 分别统计文本、图片、文件、浏览器卡片
2. 按 updatedAt 或 createdAt 倒序看最近内容
3. 告诉我最近新增了哪些对象
4. 用中文汇总，不要改文件
```

### 2.5 模糊检索

```text
我记不清具体内容了，请帮我做一次模糊检索。

关键词：
- 客户
- 微信
- 账号
- 登录

要求：
1. 同时查文本记录、浏览器卡片、文件资源备注
2. 输出每类命中的数量
3. 按相关度给我列出前 10 条
4. 不要修改文件
```

## 3. 增：新增类示例

注意：新增前要先让 AI 说明准备写到哪里。

### 3.1 新增一条文本记录

```text
现在请你在 NextVault 里新增一条文本记录。

内容：
今天和客户确认了报价方案，等待对方回执。

要求：
1. 先告诉我你准备写入哪个目录、哪个文件
2. 再执行新增
3. 写入后告诉我新对象的 id、文件路径、创建时间
4. 不要动其他已有数据
```

### 3.2 新增一条带备注的文本记录

```text
请在 NextVault 里新增一条文本记录，并附带备注。

正文：
备用服务器登录地址已经迁移到新网关。

备注：
这是运维信息，后续还要补 SSH 端口。

要求：
1. 先说明要写哪个 JSON 文件
2. 执行新增
3. 返回新增结果
```

### 3.3 新增一个浏览器卡片

```text
请在 NextVault 里新增一个浏览器卡片。

信息：
- name: GitHub 工作账号
- domain: github.com
- openUrl: https://github.com/login
- username: demo_user
- password: demo_password
- remark: 这是演示卡片

要求：
1. 先告诉我会写入哪个卡片文件
2. 再执行
3. 写完后告诉我文件路径和对象 id
```

### 3.4 新增一个文件资源记录

```text
请在 NextVault 中新增一个文件资源记录。

源路径：
C:\Users\Asus\Desktop\demo.txt

模式：
link

备注：
这是测试资源

要求：
1. 先检查源路径是否存在
2. 说明准备写入哪个对象文件
3. 执行新增
4. 返回结果
```

## 4. 改：修改类示例

### 4.1 修改文本记录正文

```text
请帮我找到包含“报价方案”的那条文本记录，并把正文改成：

今天和客户确认了最新报价方案，已经进入待签约阶段。

要求：
1. 先定位要改的对象
2. 把原内容展示给我确认
3. 再修改正文
4. 告诉我修改了哪个文件
```

### 4.2 修改文本记录备注

```text
请找到包含“备用服务器登录地址”的记录，把备注改成：

这是重要运维信息，必须同步到内部文档。

要求：
1. 先查出对应对象
2. 只修改备注，不改正文
3. 返回修改后的结果
```

### 4.3 修改浏览器卡片

```text
请找到 domain 为 `github.com` 的浏览器卡片。

把它改成：
- name: GitHub 主账号
- username: main_account
- password: new_password_123
- remark: 这是当前正式使用的账号

要求：
1. 先展示原始值
2. 再执行修改
3. 告诉我修改的是哪个文件
```

### 4.4 修改文件资源备注

```text
请找到名字里包含 `demo.txt` 的文件资源，把备注改成：

这个文件只是联调用的测试样例。

要求：
1. 先查出对象
2. 只改 note
3. 返回修改前后对比
```

### 4.5 批量修改

```text
请把 NextVault 里所有备注为空、但名字包含“测试”的文件资源，统一补一个备注：

历史测试数据

要求：
1. 先列出将要修改的对象
2. 等我确认后再修改
3. 修改后输出修改清单
```

## 5. 删：删除类示例

删除最危险，必须先列出目标。

### 5.1 删除单条文本记录

```text
请帮我删除包含“等待对方回执”的那条文本记录。

要求：
1. 先查出对应对象
2. 把命中的内容给我确认
3. 确认后再删除
4. 告诉我删除了哪个文件
```

### 5.2 删除浏览器卡片

```text
请删除 domain 为 `example.com` 的浏览器卡片。

要求：
1. 先展示命中的卡片
2. 只删除这一个
3. 删除后汇报删除结果和文件路径
```

### 5.3 删除文件资源

```text
请删除名字为 `demo.txt` 的文件资源对象。

要求：
1. 先告诉我这是删除“对象记录”还是连备份文件一起删
2. 默认只删对象记录
3. 如果涉及备份文件，必须先提醒我
```

### 5.4 批量删除测试数据

```text
请找出所有 remark、name、text 里带“测试”“示例”“demo”的对象。

要求：
1. 分类型列出来
2. 不要立刻删除
3. 先生成待删除清单
4. 等我确认后再批量删除
```

## 6. 最稳妥的操作模板

如果你想让 AI 更稳，不要直接说“改”或“删”，而是用下面模板。

### 6.1 稳妥查找模板

```text
请在当前 NextVault 目录中查找与“XXX”相关的数据。

要求：
1. 只读，不修改
2. 返回命中的对象类型、文件路径、关键字段
3. 如果有多个结果，按相关度排序
4. 用中文总结
```

### 6.2 稳妥修改模板

```text
请先查找与“XXX”相关的目标对象。

要求：
1. 先不要修改
2. 先列出准备修改的对象及文件路径
3. 告诉我每个对象将怎么改
4. 等我确认后再执行
```

### 6.3 稳妥删除模板

```text
请先在当前 NextVault 中找出我要删除的对象。

要求：
1. 先生成删除清单
2. 不要立即删除
3. 对每个对象给出类型、路径、命中原因
4. 等我确认后再执行删除
```

## 7. 你以后最常用的一句

如果你懒得每次写很多，可以直接对 AI 说：

```text
你先只读扫描当前 NextVault，帮我找和“XXX”相关的数据，列出文件路径和命中内容；没有我的确认，不要增删改。
```

---

## 8. 推荐工作流程

你自己用的时候，建议固定成这 4 步：

1. 先让 AI 扫描当前目录结构
2. 再让 AI 查目标对象
3. 如果要改或删，先让它列出清单
4. 你确认后，再让它执行

这样最稳，不容易误删。
