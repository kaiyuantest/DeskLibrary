# V2 修复测试指南

## 🎯 核心改进

**V1 问题**：删除整个域名的所有 Cookie → 登录状态丢失  
**V2 方案**：只删除要替换的具体 Cookie → 登录状态保持

## 🧪 关键测试场景

### 场景 1：基本注入测试（最重要）

**目的**：验证注入后登录状态是否保持

**步骤**：
1. 创建端口 9222 的浏览器
   ```
   在应用中输入端口 9222，点击"打开新浏览器"
   ```

2. 登录网站
   ```
   访问 https://www.swdtbook.com/index.php/index/user/login
   输入账号密码
   确认登录成功，能看到个人信息
   ```

3. 关闭浏览器

4. 离线导入 Cookie
   ```
   点击"离线导入 Cookie"
   点击"开始扫描来源"
   选中端口 9222
   点击"加载 Cookie"
   选中 swdtbook.com
   点击"导入"
   ```

5. 使用卡片打开浏览器
   ```
   在卡片列表中找到 swdtbook.com
   点击"打开"按钮
   ```

**预期结果**：
- ✅ 浏览器打开
- ✅ **登录状态有效**（这是关键！）
- ✅ 可以访问个人信息
- ✅ 不会要求重新登录

**如果失败**：
- ❌ 打开后未登录 → 说明 Cookie 注入有问题
- ❌ 登录后立即失效 → 说明删除了重要的 Cookie

### 场景 2：已登录状态下注入（新增测试）

**目的**：验证在已登录的浏览器中注入不会破坏登录状态

**步骤**：
1. 打开端口 9222 的浏览器（使用卡片打开或手动打开）

2. 确认已登录
   ```
   访问 swdtbook.com
   确认能看到个人信息
   ```

3. 点击卡片的"注入"按钮
   ```
   选择"注入到当前浏览器"（如果有这个选项）
   或者再次点击"打开"按钮
   ```

4. 刷新页面

**预期结果**：
- ✅ 注入成功
- ✅ **登录状态仍然有效**（V1 会失败）
- ✅ 可以继续访问个人信息

**V1 vs V2 对比**：
- V1：❌ 登录状态丢失，需要重新登录
- V2：✅ 登录状态保持，无需重新登录

### 场景 3：登录后更新 Cookie

**目的**：验证更新 Cookie 不会破坏登录状态

**步骤**：
1. 使用卡片打开浏览器，确认登录有效

2. 在浏览器中进行一些操作（浏览页面、修改设置等）

3. 点击卡片右上角的"更新最新 Cookie"按钮

4. 等待更新完成

5. 刷新浏览器页面

**预期结果**：
- ✅ Cookie 更新成功
- ✅ **登录状态仍然有效**
- ✅ Cookie 数量合理（可能增加，这是正常的）
- ✅ 用户设置和偏好保持不变

### 场景 4：重新登录测试

**目的**：验证注入后可以正常重新登录

**步骤**：
1. 使用卡片打开浏览器

2. 点击"退出登录"

3. 重新输入账号密码登录

4. 点击个人信息或其他需要登录的页面

**预期结果**：
- ✅ 退出成功
- ✅ 重新登录成功
- ✅ **登录状态持久有效**（V1 会失败）
- ✅ 可以正常访问所有页面
- ✅ **不需要删除卡片或数据目录**

**V1 vs V2 对比**：
- V1：❌ 登录成功但立即失效，无法保存状态
- V2：✅ 登录成功且状态持久，正常使用

1. 打开开发者工具
2. 切换到 Application 标签
3. 左侧选择 Cookies
4. 查看 swdtbook.com 的 Cookie

**检查点**：

## ⚠️ 常见问题

### Q1: 注入后仍然未登录？

**可能原因**：
1. 导入的 Cookie 本身已过期
2. Cookie 的 domain 不匹配
3. 网站需要其他 Cookie（不只是 session）

**解决方法**：
1. 确保在登录状态下导入 Cookie
2. 检查 Cookie 的 domain 字段
3. 导入时选择"包含所有 Cookie"

### Q2: 登录后立即失效？

**V1 的问题**：
- 这是 V1 的已知问题
- 请确认使用的是 V2 版本（4fb4fdd）

**V2 仍有问题**：
- 检查是否有其他代码也在删除 Cookie
- 查看控制台是否有错误
- 确认网站没有特殊的安全机制

### Q3: Cookie 数量异常？

**正常情况**：
- 注入后 Cookie 数量可能增加（网站设置新 Cookie）
- 这是正常的，不是问题

**异常情况**：
- Cookie 数量减少 → 可能删除了不该删除的 Cookie
- Cookie 数量暴增 → 可能有重复的 Cookie
